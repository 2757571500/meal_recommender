# 数据存储

## 文件清单

| 文件 | 类型 | 用途 |
|------|------|------|
| `config.json` | 配置 | AI 提供商、天气代码、缓存设置、推送地址 |
| `profile.json` | 用户画像 | 常驻地、口味偏好、饮食限制、厨艺水平等 |
| `enums.json` | 枚举定义 | 口味、菜系、难度等可配置枚举值 |
| `dishes.json` | 菜品库 | 所有可用菜品及元数据（口味、菜系、难度、耗时等） |
| `cache.json` | 缓存 | 推荐历史记录，用于短期不重复 |

## config.json

```json
{
  "ai": {
    "provider": "longcat",
    "base_url": "https://api.longcat.chat/openai",
    "api_key": "sk-xxx",
    "model": "LongCat-Flash-Chat-2602-Exp"
  },
  "weather": {
    "provider": "itboy",
    "city_code": "101220101"
  },
  "cache": {
    "no_repeat_days": 7
  },
  "push": {
    "push_url": "https://push.showdoc.com.cn/server/api/push/{token}"
  }
}
```

## profile.json

用户画像，与运行配置分离，独立文件便于管理：

```json
{
  "hometown": "合肥",
  "serving_size": 2,
  "max_cook_time": 30,
  "skill_level": "普通",
  "taste": ["偏咸", "偏辣"],
  "cuisine_preferences": ["川菜", "徽菜"],
  "diet_type": "无限制",
  "avoid_ingredients": ["羊肉", "内脏"]
}
```

字段说明见 `enums.json` 枚举定义。

## enums.json

可配置的枚举值，增删选项无需改代码：

```json
{
  "taste": ["偏咸", "偏淡", "偏辣", "偏麻", "偏甜", "偏酸", "偏鲜"],
  "cuisine": ["川菜", "鲁菜", "粤菜", "苏菜", "浙菜", "闽菜", "湘菜", "徽菜", "东北菜", "西北菜", "北京菜", "上海菜", "湖北菜", "云南菜", "贵州菜", "清真菜", "素食", "西餐", "日韩", "东南亚"],
  "skill_level": ["新手", "普通", "熟练"],
  "diet_type": ["无限制", "素食", "纯素", "清真"],
  "dietary_tags": ["含辣", "含花生", "含海鲜", "含蛋", "含奶", "含麸质"],
  "dish_taste": ["咸", "淡", "辣", "麻", "甜", "酸", "鲜", "香", "清淡"]
}
```

## dishes.json

菜品数组，每个元素包含元数据字段：

```json
{
  "name": "麻婆豆腐",
  "reason": "麻辣鲜香，嫩滑入味，适合下饭。",
  "cuisine": "川菜",
  "taste": ["辣", "麻"],
  "difficulty": "普通",
  "prep_time": 20,
  "ingredients": ["豆腐", "牛肉末", "豆瓣酱", "花椒", "蒜苗"],
  "diet_type": "无限制",
  "dietary_tags": ["含辣"]
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 菜名 |
| `reason` | 是 | 推荐理由 |
| `cuisine` | 是 | 菜系（从 enums.json 取值） |
| `taste` | 是 | 口味标签数组（从 enums.json dish_taste 取值） |
| `difficulty` | 是 | 难度：新手/普通/熟练 |
| `prep_time` | 是 | 烹饪耗时（分钟） |
| `ingredients` | 是 | 主要食材数组 |
| `diet_type` | 是 | 饮食类型：无限制/素食/纯素/清真 |
| `dietary_tags` | 否 | 特殊标签数组（从 enums.json dietary_tags 取值） |

按 `name.strip().lower()` 去重，同名菜品保留第一条，新推荐可更新 reason。

## cache.json

```json
{
  "records": [
    {"name": "麻婆豆腐", "date": "2026-04-30", "meal": "午餐"},
    {"name": "番茄蛋汤", "date": "2026-04-30", "meal": "午餐"}
  ]
}
```

每条记录包含菜名、推荐日期和餐段。`get_excluded_dishes(days)` 根据日期过滤 N 天内记录，组装到 prompt 中让 AI 避开。
