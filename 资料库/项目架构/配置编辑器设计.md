# 配置网页编辑器设计

## 概述

一个零依赖的极简网页配置工具，使用 Python 内置 `http.server` 实现。运行后在本机提供 HTTP 服务，通过表单界面可视化编辑 `config.json`、`profile.json`、`enums.json`，保存后直接写盘。

适用场景：青龙面板等无图形界面的服务器环境，通过浏览器远程配置。

## 技术选型

- **Python 内置 `http.server`** — 零外部依赖，项目只有 `requests` 一个包
- **纯 HTML/CSS/JS 内嵌** — 所有前端代码以字符串形式写在 Python 中，单文件部署
- **GET/POST API** — 页面加载用 GET，提交表单用 POST JSON

## 文件

`src/config_editor.py` — 单文件，包含 HTTP 服务和前端页面。

## 启动方式

```bash
python src/config_editor.py
# 默认监听 0.0.0.0:8080，可通过 -p 参数指定端口
```

启动后输出：
```
配置编辑器已启动，请访问 http://0.0.0.0:8080
按 Ctrl+C 停止服务
```

### 交互逻辑

- 折叠面板：点击标题栏展开/收起，默认展开"运行配置"
- 文本输入框：直接编辑单行文本
- 下拉选择框：profile 中单值字段（skill_level、diet_type）
- 多选框：profile 中多值字段（taste、cuisine_preferences），**选项从 enums.json 读取**
- 逗号文本：enums 中的列表字段，用逗号分隔编辑
- 所有修改在本地缓存，点击右上角"保存配置"统一提交

## API 接口

### GET / — 返回主页面 HTML

### GET /api/config — 读取当前配置

返回三个文件的 JSON 合并：
```json
{
  "config": { "ai": { "base_url": "...", ... }, "push": { ... } },
  "profile": { "hometown": "合肥", ... },
  "enums": { "taste": [...], "cuisine": [...], ... }
}
```

### POST /api/config — 保存配置

请求体：
```json
{
  "config": { ... },
  "profile": { ... },
  "enums": { ... }
}
```

分别写入 `data/config.json`、`data/profile.json`、`data/enums.json`。
返回 `{"status": "ok"}` 或 `{"status": "error", "message": "..."}`。

## 注意点

- config.json 包含 API Key，页面中密码输入框显示（****），但传输仍为明文（内网环境）
- 写入时保持 JSON 缩进格式（indent=2）
- 文件写失败时返回错误信息，不覆盖原文件
- enums 中的 taste 和 cuisine 变更后，下次刷新页面 profile 的多选框选项同步更新
- enums 中的 diet_type 变更后，profile 的下拉选项同步更新
