## 模块职责

### main.py — 推荐入口
- 青龙面板定时任务，cron `30 8 * * *`
- 编排全流程：加载配置 + 画像 + 枚举 → 加载菜品库 → 获取天气 → 过滤菜品 → AI 推荐 → 推送结果
- 检测到菜品库为空时打印提示并跳过，不推送失败通知
- AI 调用失败时推送失败通知并 exit(1)

### update_library.py — 菜品发现入口
- 青龙面板定时任务，cron `30 6 * * 1`
- 编排：加载配置 + 画像 + 枚举 → 获取天气 → AI 发现新菜（JSON 数组格式） → 枚举校验 → 入库 → 推送结果
- 与推荐完全独立，可单独调试和调度
- 清空 dishes.json 后需手动跑一次填充菜品库

### recommender.py — 推荐引擎
- `recommend()`：从已有菜品库选菜推荐，更新缓存
- `discover_dishes()`：调用 AI 发现新菜，自动入库
- `_build_prompt()`：组装 AI prompt，包含时令、天气、菜品库、排除列表、两地信息（天气城市 + 常驻地）、就餐人数、口味偏好、菜系偏好、忌口食材、厨艺水平
- `_build_discover_prompt()`：组装菜品发现 prompt，要求 AI 返回 JSON 数组，字段值严格从枚举中取
- `filter_dishes(dishes, profile)`：根据用户画像对菜品库做硬过滤
  - diet_type：用户无限制则全部保留；有限制则保留"匹配 + 无限制"
  - 难度：dish.difficulty 等级 ≤ profile.skill_level
  - 耗时：dish.prep_time ≤ profile.max_cook_time
- `_parse_response()`：解析 AI 返回的三餐推荐文本
- `_parse_dish_list()`：解析 AI 返回的 JSON 数组格式菜品列表
- `_format_output()`：格式化控制台输出
- `get_season()`：根据月份返回季节

### ai_client.py — AI 客户端
- 封装 OpenAI Chat Completions 兼容接口
- 支持任意兼容 API（LongCat、DeepSeek、OpenAI 等）
- 切换模型只需改 config.json，不改代码

### dish_manager.py — 菜品库管理
- `load_dishes()` / `save_dishes()`：JSON 文件读写
- `add_new_dishes(enums)`：新增或更新菜品（同名更新理由），入库前调用 `validate_dish()` 校验
- `validate_dish(dish, enums)`：校验单条菜品字段值是否在 enums 允许范围内，超范围丢弃并打印告警，不阻塞其他有效菜品入库
- `list_dish_names()`：返回所有菜名列表
- `deduplicate()`：合并同名菜品（忽略大小写和空格）

### cache_manager.py — 缓存管理
- `get_excluded_dishes(days)`：返回 N 天内推荐过的菜品集合
- `add_record()`：记录本次推荐菜品到缓存
- `cleanup(days)`：清理过期记录

### weather.py — 天气客户端
- 调用 itboy 天气 API
- 失败降级返回 `("未知", "未知")`

### push.py — 推送客户端
- 通过 ShowDoc API 推送到微信
- push_url 为空时跳过
- 异常时打印日志，不崩溃

### config.py — 配置加载
- `load_config()`：读取 config.json，转为 SimpleNamespace
- `load_profile()`：读取 profile.json，转为 SimpleNamespace
- `load_enums()`：读取 enums.json
- 三文件分离：运行配置、用户画像、枚举定义各司其职

## 数据流

单向管线：**配置 + 画像 → 加载菜品 → 天气 → 硬过滤 → AI 推理 → 推送**

- 推送标题城市来源：`profile.hometown`
- 天气城市来源：`config.weather.city_code`（可与常驻地不同，覆盖出差场景）
- 菜品库为空时不执行推荐，仅打印提示
- 任何外部依赖失败都不应阻塞整体流程，而是降级 + 通知

## 部署架构

青龙面板定时任务环境：
- 无交互输入，纯脚本执行
- 所有数据存本地 JSON 文件，无数据库
- 仅依赖 `requests` 包
