# 需求变更：配置字段归并到用户画像

## 变更原因

`weather.city_code`（天气查询城市）和 `cache.no_repeat_days`（菜品不重复天数）本质上是用户维度的设置，与 AI 提供商、推送地址等运行配置无关。放在 `config.json` 中语义不合理，归入 `profile.json` 更符合"用户画像"的定位。

## 变更内容

### 字段迁移

| 字段 | 原位置 | 新位置 |
|------|--------|--------|
| `weather.city_code` | `config.json` → `config.weather.city_code` | `profile.json` → `profile.weather_city_code` |
| `cache.no_repeat_days` | `config.json` → `config.cache.no_repeat_days` | `profile.json` → `profile.no_repeat_days` |

### 数据文件变化

**config.json**（移除 weather、cache 段）：
```json
{
  "ai": { "provider": "longcat", "base_url": "...", "api_key": "...", "model": "..." },
  "push": { "push_url": "..." }
}
```

**profile.json**（新增两字段）：
```json
{
  "hometown": "合肥",
  "weather_city_code": "101220101",
  "no_repeat_days": 7,
  "serving_size": 2,
  ...
}
```

### 代码引用变化

```
# 改前
config.weather.city_code
config.cache.no_repeat_days

# 改后
profile.weather_city_code
profile.no_repeat_days
```

涉及文件：`recommend_daily.py`、`update_library.py`、`recommender.py`

### config.py 调整

`load_config()` 不再加载 weather 和 cache 段，`load_profile()` 返回的 SimpleNamespace 包含新字段。

## 影响范围

- `config.json` 不再需要 weather 和 cache 配置，更精简
- 天气城市和重复天数随用户画像走，一个 profile 对应一套设置
- 向后兼容：旧 config.json 中残留 weather/cache 段不会导致报错（json.load 忽略多余字段）
- 所有 82 个测试通过
