# 需求变更：新增配置网页编辑器

## 变更原因

此前修改配置需手动编辑 JSON 文件，容易格式错误。需要一个可视化工具来安全便捷地修改运行配置、用户画像和枚举值。

## 变更内容

### 新增文件

| 文件 | 说明 |
|------|------|
| `src/config_editor.py` | 基于 `http.server` 的零依赖网页配置编辑器 |
| `资料库/项目架构/配置编辑器设计.md` | 设计文档 |

### 功能

- 监听 `0.0.0.0:8089`（可通过 `-p` 参数指定端口）
- 启动后自动在浏览器中打开编辑页面
- 三个折叠面板：

**运行配置**（config.json）
- AI 接口地址、API Key、AI 模型、推送地址

**用户画像**（profile.json）
- 文本框：常驻地、天气城市码、不重复天数、人数、烹饪时长、忌口食材
- 下拉框：厨艺水平（选项从 enums.skill_level 读取）、饮食类型（选项从 enums.diet_type 读取）
- 多选框：口味偏好（选项从 enums.taste 读取）、菜系偏好（选项从 enums.cuisine 读取）
- 忌口食材使用标签式输入

**可选值管理**（enums.json）
- 所有枚举字段使用标签式输入（Tag Input），点击 + 添加，点击 × 删除
- 修改后刷新页面，用户画像中的选项即时同步

### API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 返回编辑页面 HTML |
| GET | `/api/config` | 返回 `{config, profile, enums}` 合并 JSON |
| POST | `/api/config` | 接收 `{config, profile, enums}`，分别写入三个文件 |

## 设计要点

- **零外部依赖**：只使用 Python 内置 `http.server`，项目仅需 `requests` 包
- **单文件部署**：HTML/CSS/JS 全部内嵌在 Python 字符串中，一个文件即完整应用
- **前端动态渲染**：JS 获取配置后在前端渲染表单，枚举值联动即时生效

## 影响范围

- 无业务逻辑变更
- 配置文件结构不变
- config.json（含 API Key）仍保持 `.gitignore` 不被提交
