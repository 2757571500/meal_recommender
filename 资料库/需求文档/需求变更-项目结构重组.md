# 需求变更：项目结构重组

## 变更原因

原有项目中 Python 源码和 JSON 数据文件混放在根目录，结构扁平、层次不清。两个入口文件（main.py、update_library.py）与其他业务模块同级，难以区分调度层与逻辑层。

## 变更内容

### 目录结构

改前：
```
meal_recommender/
├── main.py, *.py, *.json    ← 全部混放
├── tests/
└── 资料库/
```

改后：
```
meal_recommender/
├── src/
│   ├── recommend_daily.py       ← 入口：每日推荐（原 main.py）
│   ├── update_library.py        ← 入口：菜品库更新
│   └── core/                    ← 业务逻辑层
│       ├── config.py
│       ├── recommender.py
│       ├── ai_client.py
│       ├── dish_manager.py
│       ├── cache_manager.py
│       ├── weather.py
│       └── push.py
├── data/                        ← JSON 数据文件
│   ├── config.json
│   ├── profile.json
│   ├── enums.json
│   ├── dishes.json
│   └── cache.json
├── tests/
├── 资料库/
└── CLAUDE.md
```

### 主要改动

| 文件 | 变更 |
|------|------|
| `main.py` → `src/recommend_daily.py` | 更名并移入 src/，cron 注释同步更新 |
| `update_library.py` → `src/update_library.py` | 移入 src/，不变 |
| `*.py` → `src/core/*.py` | 7 个业务模块移入 core/ 子目录 |
| `*.json` → `data/*.json` | 5 个数据文件集中到 data/ |
| `tests/*.py` | 所有 import 路径和 mock patch 路径更新为 `core.*` 前缀 |
| `conftest.py` | 新增 `patch_dishes_path` fixture 隔离文件操作 |
| `.gitignore` | 路径更新 |

### 导入路径变化

```
# 改前
from config import load_config
from dish_manager import load_dishes
from ai_client import AIClient

# 改后
from core.config import load_config
from core.dish_manager import load_dishes
from core.ai_client import AIClient
```

### 入口执行

```
# 改前
python main.py
python update_library.py

# 改后
python src/recommend_daily.py
python src/update_library.py
```

## 设计思想

- **两层分离**：`src/` 入口文件负责编排调用（调度层），`src/core/` 模块负责具体业务逻辑
- **数据与代码分离**：JSON 数据文件集中在 `data/`，与代码物理隔离
- **git 识别重命名**：使用 `git mv` 操作，保留文件历史
- **测试隔离**：新增 `patch_dishes_path` fixture，将涉及文件写的测试重定向到临时路径，不污染真实数据文件

## 影响范围

- 执行命令需加上 `src/` 前缀
- 所有 Python 导入路径增加 `core.` 前缀
- 测试 mock patch 路径从 `"ai_client.requests.post"` 变为 `"core.ai_client.requests.post"`
- 无业务逻辑变更，推荐流程和菜品发现行为不变
