# 编程规范

## 命名规范

| 元素 | 规范 | 示例 |
|------|------|------|
| 函数/变量 | snake_case | `get_excluded_dishes()`, `no_repeat_days` |
| 类名 | PascalCase | `AIClient` |
| 常量 | UPPER_SNAKE_CASE |（本项目暂无） |
| 私有函数 | 下划线前缀 | `_build_prompt()`, `_parse_response()` |
| 文件/模块 | snake_case | `dish_manager.py`, `cache_manager.py` |

## 设计原则

### 健壮性优先
- 所有外部依赖（天气 API、AI API、推送接口）调用必须包裹 try-except
- 失败时降级处理，不允许未捕获异常导致脚本崩溃
- 关键路径失败必须推送通知，不能静默失败

### 配置驱动
- 所有可变参数放在 `config.json`，代码不做硬编码
- AI 提供商、模型、城市、推送地址等均可通过配置文件切换

### 内聚
- 工具类函数放在对应模块，不集中到单个 util 文件
- 各模块职责单一，不跨模块引用内部函数

## 依赖管理

- 最小依赖原则，尽量使用 Python 标准库
- 目前唯一外部依赖：`requests`（HTTP 请求）
- 无包管理文件（requirements.txt），部署时手动 pip install

## 异常处理模式

```python
try:
    resp = requests.get(url, timeout=10)
    resp.raise_for_status()
    return parse_result(resp)
except Exception as e:
    print(f"操作失败: {e}")
    return fallback_value  # 不 raise
```

所有模块遵循同一种异常处理模式：捕获 → 打印日志 → 返回降级值。

## 代码注释规范

- 公有函数和类写 docstring，说明用途、参数、返回值
- 注释用中文写
- 不写多余的"这段代码做什么"式注释——好的命名已经表达了

## 与青龙面板兼容

- 脚本文件头部标注 cron 表达式
- 纯脚本执行，无交互输入
- 执行结果通过推送通知而非终端输出触达用户
